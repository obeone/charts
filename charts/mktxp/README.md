## About

[MKTXP](https://github.com/akpw/mktxp) is a Prometheus exporter for MikroTik RouterOS devices. It collects
metrics from one or several routers (interfaces, wireless, CAPsMAN, DHCP,
firewall, system health, ...) and pairs nicely with its companion Grafana
dashboard.

- **Application:** [github.com/akpw/mktxp](https://github.com/akpw/mktxp)
- **Container image:** [`ghcr.io/akpw/mktxp`](https://github.com/akpw/mktxp/pkgs/container/mktxp)
- **Chart source:** [`charts/mktxp`](https://github.com/obeone/charts/tree/main/charts/mktxp)

This chart is built on the [k8s-at-home common library](https://github.com/k8s-at-home/library-charts)

Defaults are meant to work out of the box on any cluster. The full list of


### Declaring routers

Routers to monitor are declared in the bundled ConfigMap:

```yaml
configmap:
  config:
    enabled: true
    data:
      mktxp.conf: |
        [my-router]
            hostname = 192.0.2.1
            username = mktxp
            password = change-me
```

See the [upstream documentation](https://github.com/akpw/mktxp#mikrotik-device-config)
for the RouterOS side of the setup (dedicated read-only user, API access).

### Prometheus integration

With `metrics.enabled` set (the default), the chart creates a `ServiceMonitor`
for the Prometheus Operator; scrape interval and labels are tunable under
`metrics.serviceMonitor`.

## Verifying the chart signature

Charts in this repository are signed with GPG and every release ships a
provenance file. The public key is available at
[`charts.obeone.cloud/public_key.gpg`](https://charts.obeone.cloud/public_key.gpg),
fingerprint `B9FE852F28888D27F8C9A11CD33E04CD22E335CE`.

```shell
# Import the signing key into a legacy keyring (helm verifies with GnuPG v1 keyrings)
curl -fsSL https://charts.obeone.cloud/public_key.gpg | gpg --import
gpg --export > ~/.gnupg/pubring.gpg

# Pull the chart and check it against its provenance file
helm pull --verify obeone/mktxp
```
