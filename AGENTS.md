# Contributor Guide

## Testing rules

- If you change anything in a `Chart.yaml`, run `helm dep up` to refresh the dependency lock file.
- Whatever you change, always run `helm lint` on the affected chart and fix every error it reports before committing.

## Documentation

- Chart `README.md` files are generated by helm-docs: never edit them by hand. Edit `charts/_templates.gotmpl` (the shared skeleton) or the chart's own `README.md.gotmpl`, then regenerate from the repository root:

  ```shell
  helm-docs --chart-search-root=charts --template-files=./_templates.gotmpl --template-files=README.md.gotmpl
  ```

- After any change to `values.yaml` (the `# --` comments feed the Values table) or `Chart.yaml` (version, description), re-run helm-docs so the `README.md` stays in sync, and commit the regenerated README alongside the rest.

- The root `README.md` has a table whose `App` column mirrors each chart's `appVersion`. It is resynced by `scripts/sync-readme-versions.py`; do not edit it by hand.

## Pre-commit hooks

The repository ships a `.pre-commit-config.yaml` that automates the two points above. Install it once:

```shell
pre-commit install
```

Local requirements: `helm-docs` (`brew install norwoodj/tap/helm-docs`) and `python3`. On every commit, two hooks run:

- `helm-docs` regenerates the chart `README.md` files from the shared skeleton.
- `sync-readme-versions` realigns the `App` column of the root `README.md` with each chart's `appVersion`.

Both rewrite files in place and fail the commit when they change something: just re-`git add` and commit again.
